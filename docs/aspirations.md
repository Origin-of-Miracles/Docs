# 我们的憧憬 | Our Aspirations

> "空有‘外壳’，没有实质。只有空荡荡的一片学院建筑群，却见不到一个学生……"
>
> 我们不希望基沃托斯在方块世界里只是一座死城。

## 1. 拒绝孤独的“布景” (Beyond the Shell)

现在的很多还原地图，建筑再精美，走进去只有空荡荡的回廊。那种**“物是人非”的孤独感**往往比还原度本身更让人难受。

我们想要的，是一个 **“活着”的基沃托斯**。

## 2. 真正的“学生” (Living Students)

我们不满足于冷冰冰的 GUI 交互。

- **YSM 3D 兼容**: 每一个学生都应该拥有高精度的 3D 模型（兼容 Yes Steve Model），她们会走动，会看你，会有表情。
- **实体交互**: 靠近爱丽丝，右键点击她，不是打开一个菜单，而是直接开始对话。
- **“人多才好玩” (Context-Aware Group Chat)**:
    - 当你走进【游戏开发部】的活动室，你不是在和单独的一个 NPC 说话。
    - 这是一个**场域 (Field)**。爱丽丝、桃、绿、柚子都在这里。
    - 你的对话会被整个环境感知。你问一句“今天玩什么？”，桃可能会抢答，绿会吐槽，爱丽丝会附和。

## 3. 无限可能的生活 (Procedural Life)

拒绝枯燥的重复任务。我们引入 **LLM (大语言模型)** 作为世界的“地下城主 (DM)”。

- **动态委托系统**:
    - 任务不再是写死的脚本。
    - LLM 会根据当前的时间、天气、好感度，生成独一无二的日常/周常委托。
    - **合法操作集 (Action Space)**: 我们提供 API 给 LLM，确保它生成的任务（如“去清理5个僵尸”、“帮忙找回丢失的游戏卡带”）在游戏机制上是可执行的。
- **拒绝脚本感**: 每一天的对话，每一天的任务，都是新鲜的。

## 4. 灵魂的共鸣 (Voice & Soul)

- **真实语音**: 接入 GPT-V / VITS 技术。
- 当爱丽丝高兴时，你能听到她欢快的语调；当她失落时，你能听到她的低语。
- 配合 3D 模型的动作与口型，打破次元壁。

**让爱丽丝，和所有的学生们，在这里真正地生活下去。**
